## 🧩 Project Overview

The project simulates a **simple student record system**.

### Features

- Add a student
- Display student details
- Remove a student
- View all records

The system uses:

- **ArrayList** for storing students
- **Classes and methods** for modular design

This type of program is often used as a **beginner-level Java project** to demonstrate understanding of OOP.

## 🎯 Interview Preparation (Day 30 Level)

**Q1. Why are mini projects important when learning programming?**  
They help apply theoretical concepts in real scenarios.

**Q2. Why use ArrayList in this project?**  
Because it provides dynamic storage for student objects.

**Q3. What OOP concepts were used in this project?**  
Encapsulation, classes, and object interaction.

**Q4. How can this project be improved?**  
By adding file storage, search functionality, and better validation.

**Q5. What is the benefit of modular code design?**  
It improves readability, maintainability, and scalability.
