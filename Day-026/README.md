## 🧩 Understanding Encapsulation & Access Control

Encapsulation means:

- Wrapping data and methods into a single unit
- Restricting direct access to variables
- Providing controlled access through methods

Access modifiers define:

- Who can access variables, methods, and classes
- How visibility works inside and outside packages

This is essential for:

- Secure coding
- API development
- Large-scale system design

## 🎯 Interview Preparation (Day 26 Level)

**Q1. What is encapsulation?**  
Encapsulation is wrapping data and methods together and restricting direct access to data.

**Q2. Why are variables kept private?**  
To protect data from unauthorized modification.

**Q3. Name the four access modifiers in Java.**  
private, default, protected, public.

**Q4. What is the difference between protected and default?**  
Protected allows access in subclasses even in different packages, while default is package-level access only.

**Q5. How does encapsulation improve security?**  
By allowing validation and controlled access to class data.
